## 考核架構

**考核原則：** 只靠 Claude Code 本體 + 本地檔案 + 程式執行，不碰外部服務。
考核重點：提示詞設計、產出可重現性、把好做法抽成 Skill 的能力。

**節奏：** 前期全做「純」的（核心肌肉），中後期進「非純」的，每串一個工具教一個觀念。

## 育：個人健身／習慣計畫產生器

**替換原因：** 原本「課文→出題」功能性太單一；健身計畫產生器輸出結構更豐富，更能考核「可執行結構 vs 泛泛建議」的提示詞設計能力。

**考核亮點：** 輸出必須是可執行的結構（週表格、打卡清單），不是模糊建議。

```text
角色一：計畫設計師（Planner）
輸入：目標（增肌/減脂/習慣養成）+ 現有能力 + 每週可用時間
輸出：週計畫 JSON（含每日項目、組數、時長）

角色二：打卡系統（Tracker）
輸入：週計畫 + 今日日期
輸出：今日打卡清單 + 完成率統計
```

## Demo 說明區塊

**目標：** 建立完整考核藍圖，並確認「育」類別改為健身/習慣計畫產生器。

**結果：** 課表確立，共 10 個純本地實驗 + 部署加考。
